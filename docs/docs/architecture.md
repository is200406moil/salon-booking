# Архитектура системы

## Общая структура
Система построена по трёхуровневой архитектуре (Presentation → Application → Data)
и реализована в рамках MVT-архитектуры Django.

## Компоненты
- **Presentation**: HTML5 + Bootstrap 5, шаблоны Django.
- **Application**: бизнес-логика в views/forms, проверка доступности слотов.
- **Data**: PostgreSQL, модели Django ORM.

## Приложения Django
- `accounts`: пользователи, роли, регистрация/авторизация.
- `services`: каталог услуг.
- `masters`: мастера и рабочие слоты.
- `appointments`: записи клиентов и личный кабинет.

## Логическая модель данных
Используются сущности `User`, `Service`, `MasterProfile`, `WorkingSlot`, `Appointment`.
Связи реализованы внешними ключами и M2M отношениями согласно отчёту ПР 5-8.

## Информационные потоки
- Клиент выбирает услугу и слот → Application проверяет доступность → Data сохраняет запись.
- Мастер получает расписание → Application формирует выборку → Presentation отображает календарь.

## Диаграммы
DFD и IDEF0 диаграммы приведены в отчётах ПР 5-8 и описывают процессы записи,
управления услугами и расписанием.
