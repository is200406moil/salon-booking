# Модели данных

## Основные сущности

- **User** — пользователь системы (клиент, мастер, администратор), расширяет стандартный
  пользователь Django и содержит роль и телефон.
- **Service** — услуга салона (название, описание, длительность, цена, активность).
- **MasterProfile** — профиль мастера с набором оказываемых услуг.
- **WorkingSlot** — рабочий слот мастера (начало, окончание, доступность).
- **Appointment** — запись клиента на услугу в выбранный слот.

## Связи

- `User` ↔ `MasterProfile` — связь один-к-одному.
- `MasterProfile` ↔ `Service` — многие-ко-многим.
- `MasterProfile` ↔ `WorkingSlot` — один-ко-многим.
- `Appointment` связывает клиента, мастера, услугу и слот.

## Ограничения

- Запись привязана к одному слоту и одному мастеру.
- Слот становится недоступен после бронирования.
- Услуги и мастера имеют флаг активности, скрывающий их из клиентского списка.
